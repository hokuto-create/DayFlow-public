# 市販教材テンプレート

教材ビューの「テンプレートから追加」で使う、市販の参考書・問題集のプリセットデータです。
アプリ本体（index.html）には埋め込まず、選択時に fetch で遅延読み込みされます。
`file://` 直開き等で fetch できない環境では、テンプレ機能のUIだけが自動的に非表示になります。

## ファイル構成

- `index.json` — マニフェスト（一覧画面用のメタデータのみ）
- `<id>.json` — 教材ごとの本体データ（章・問題リスト）

## スキーマ

### index.json

```json
{
  "version": 1,
  "templates": [
    {
      "id": "chart-blue-1a",          // 本体ファイル名（<id>.json）と一致させる
      "name": "青チャート 数学I+A（増補改訂版）",
      "subject": "数学",               // アプリの「科目」名。無ければ取り込み時に自動作成される
      "publisher": "数研出版",
      "edition": "増補改訂版",
      "itemCount": 233,                // 本体の全 items 数と一致させる
      "verified": false                // 実書と照合済みなら true
    }
  ]
}
```

### 教材本体（`<id>.json`）

```json
{
  "id": "chart-blue-1a",
  "name": "青チャート 数学I+A（増補改訂版）",
  "subject": "数学",
  "publisher": "数研出版",
  "edition": "増補改訂版",
  "color": "#4A90D9",                  // 教材タブの色
  "verified": false,                   // 未確認の間は false のまま
  "reviewNote": "要確認: ...",          // 人間が確認すべき点
  "chapters": [
    {
      "title": "第1章 数と式",          // 取り込み時に各問題 text の接頭辞になる
      "note": "要確認: ...",            // 章単位の確認メモ（アプリは未使用）
      "items": [
        { "text": "例題1", "minutes": 20 }   // minutes は任意（想定所要時間・分）
      ]
    }
  ]
}
```

取り込み時の動作: `subject` が科目一覧に無ければ追加 → 教材（subjects）を1件作成 →
`chapters[].items` を問題（subjectTodos）に展開（text は「章タイトル + 半角スペース + item.text」）。

## 問題の区分方針（単語帳・一問一答など）

1問=1TODO（1 item）にするのは、1問に数分〜数十分かかる演習書（チャート、問題精講など）のみ。
単語帳・一問一答のように1項目が数秒〜数十秒で終わるものは、**章・節・番号範囲・ページ範囲を
1つの item に区分**する。

- 例: `Section 1（No.1〜100）`、`第1章 古代（問1〜80）`、`第2章 §3（p.45〜60）`
- 1語・1問単位の item は作らない。著作権上本文を載せられないため無意味な行になる上、
  粒度が細かすぎて計画アプリの管轄外（カード単位の暗記は赤シートや暗記アプリの領分）
- 「1周目」「2周目」のような周回 item も作らない（周回をこなすこと自体が目的化するため）。
  範囲を学習したら理解度マーク（✕△○◎）で定着度をセルフチェックし、
  既存の間隔反復（✕→翌日 / △→3日後 / ○→1週間後）で再浮上させる運用とする
- `minutes` にはその範囲を1回学習する想定時間（分）を入れる

## 著作権上の制約（厳守）

収録してよいのは以下のみ:

- 書名・出版社・版
- 章タイトル（目次レベル）
- 問題番号・問題の見出し語

**問題文・解答・解説の本文は一切含めないこと。** 単語帳の場合、収録単語そのものも
含めない（Section番号・単語番号の範囲のみ）。

## ⚠ 要確認（人間側のタスク）

現在の5冊はすべて `verified: false` の**プレースホルダ**です。章構成・問題数・番号は
実書と照合できていないため、リリース前に実物の目次と突き合わせて修正してください。

| ファイル | 確認事項 |
|---|---|
| `chart-blue-1a.json` | 章構成（指導要領の単元名で仮置き）、例題数・通し番号、基本例題/重要例題の区別 |
| `target-1900.json` | 6訂版のPart名・Section区切り（100語×19で仮置き） |
| `systan-5.json` | 5訂版の各Stageの単語番号範囲・収録語数（仮置き）、多義語の扱い |
| `physics-essence-mechanics.json` | 章タイトル・各章の問題数（単元名で仮置き）、EX/問の番号体系、波動編の追加要否 |
| `kisomon-seiko-1a.json` | 最新版の確認（五訂版で仮置き）、章構成・基礎問数・通し番号 |

確認・修正が済んだら、本体とマニフェスト両方の `verified` を `true` にし、
`itemCount` を実際の items 数に合わせてください。
